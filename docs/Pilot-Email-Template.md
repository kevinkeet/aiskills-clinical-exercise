# Pilot recruitment email — template

Universal template. Two things to fill in per recipient:
- `[Name]` — their first name
- `[X]` — the pilot number you assign

Reminder on arm assignment:
- **Odd codes** (Pilot1, Pilot3, Pilot5, …) → AI chat assistant. No extra login.
- **Even codes** (Pilot2, Pilot4, Pilot6, …) → UpToDate via the Stanford
  Lane Library proxy. The recipient needs an active Stanford SUNet ID.
- If a pilot isn't Stanford-affiliated, give them an odd code so they get
  the AI arm instead.

---

**Subject:** Pilot test — Stanford clinical reasoning study (~30–45 min)

Hi [Name],

I'm piloting an online clinical reasoning exercise for an IRB-approved
Stanford study (IRB-86737), and I'd really value your eyes on it before
we open recruitment to residents.

**What it is.** A single online session — roughly 30 to 45 minutes — built
around a fictional patient case. You'll work through 5 short free-response
tasks and a 22-question knowledge assessment. Depending on the code below,
you'll have either an AI chat assistant or UpToDate available alongside
the case.

**How to participate.**

1. Open https://aiskills.kevinkeet.com on a laptop or desktop (the layout
   is split-screen and doesn't fit phones).
2. When asked for your enrollment number, enter your pilot code: **Pilot[X]**
3. Walk through the exercise as you would in a real session.
4. **Leave feedback in the amber "Pilot feedback" box** that appears below
   each task and each quiz question — typos, confusing wording, wrong
   answer keys, anything you'd change. The box auto-saves as you type.

**Two things worth knowing.**

- Odd-numbered codes get the AI chat assistant (built in, no login).
  Even-numbered codes get UpToDate via the Stanford Lane Library proxy —
  you'll be asked to sign in with your SUNet ID.
- The knowledge assessment is normally "no external resources." For
  piloting, do whatever's most useful — answer as a resident would, or
  look things up to evaluate the questions. Either is helpful; just note
  what you did in the feedback box.

**Resuming.** Re-entering the same code in the same browser picks up
where you left off. Please don't share your code — once a code completes
the study, it's locked.

Bugs, confusion, or just thoughts on the design — reply to this email any
time.

Thanks for piloting,
Kevin
